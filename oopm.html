Object-Oriented Programming Methodology (OOPM)

Complete Overview for Students & Projects

What is OOPM?

Object-Oriented Programming Methodology (OOPM) is a programming approach that focuses on objects which represent real-world entities. Each object contains data (variables) and behavior (methods).

Main Concepts of OOPM

  • Class
  • Object
  • Encapsulation
  • Inheritance
  • Polymorphism
  • Abstraction

Features of OOPM

  • Modularity
  • Reusability
  • Flexibility
  • Security
  • Easy maintenance

Advantages of OOPM

  • Code reusability through inheritance
  • Improved maintainability
  • Better data security
  • Real-world problem solving
  • Scalable applications

Applications of OOPM

  • Web applications
  • Mobile applications
  • Game development
  • Enterprise software
  • GUI-based systems

Examples of OOP Languages

  • Java
  • C++
  • Python
  • C#
  • Ruby

1. Class & Object

A class is a blueprint used to create objects. An object is an instance of a class. 


class Student {
    int id;
    String name;

    void display() {
        System.out.println(id + " " + name);
    }
}

class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.id = 1;
        s.name = "Rahul";
        s.display();
    }
}

2. Constructor

A constructor initializes objects automatically when they are created. 


class Employee {
    int id;

    Employee(int i) {
        id = i;
    }
}

3. Encapsulation

Encapsulation protects data by making variables private and accessing them using methods. 


class BankAccount {
    private int balance = 1000;

    public int getBalance() {
        return balance;
    }
}

4. Inheritance

Inheritance allows a class to reuse properties of another class. 


class Animal {
    void eat() {
        System.out.println("Eating");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Barking");
    }
}

5. Polymorphism

Polymorphism allows one method to behave differently. 


class Shape {
    void draw() {
        System.out.println("Drawing shape");
    }
}

class Circle extends Shape {
    void draw() {
        System.out.println("Drawing circle");
    }
}

6. Abstraction

Abstraction hides implementation details. 


abstract class Bank {
    abstract int getInterestRate();
}

7. Interface

Interfaces support multiple inheritance. 


interface Payment {
    void pay();
}

8. Exception Handling

Handles runtime errors. 


try {
    int a = 10 / 0;
} catch (Exception e) {
    System.out.println("Error occurred");
}

9. Multithreading

Allows multiple threads to run simultaneously. 


class MyThread extends Thread {
    public void run() {
        System.out.println("Thread running");
    }
}

10. File Handling

Used to store data permanently. 


FileWriter fw = new FileWriter("data.txt");
fw.write("Hello Java");
fw.close();
